## Hello, World

Every C program starts here. K&R Chapter 1 opens with this ‚ÄĒ the smallest
complete program that does something visible.

```c
#include <stdio.h>

int main(void) {
    printf("hello, world\n");
    return 0;
}
```

What's happening:

- `#include <stdio.h>` ‚ÄĒ tells the compiler to include the standard I/O library (for `printf`)
- `int main(void)` ‚ÄĒ the entry point. Every C program starts executing here. Returns an `int` to the OS (0 = success).
- `printf("hello, world\n")` ‚ÄĒ prints the string. `\n` is a newline. Without it, the cursor stays on the same line.
- `return 0` ‚ÄĒ exit status. Convention: 0 means no error.

Compile and run:

```bash
cc hello.c -o hello
./hello
```

`cc` is the system C compiler. `-o hello` names the output binary. Without `-o`, you get `a.out`.

## Data Types

C has a small set of fundamental types. Everything else is built from these.

| Type | Size (typical) | What it holds | Example |
|------|---------------|---------------|---------|
| `char` | 1 byte | A single character (or small integer) | `'A'`, `65` |
| `int` | 4 bytes | Whole numbers | `42`, `-1`, `0` |
| `float` | 4 bytes | Decimal numbers (single precision) | `3.14f` |
| `double` | 8 bytes | Decimal numbers (double precision) | `3.14159265` |

### Modifiers

| Modifier | Meaning | Example |
|----------|---------|---------|
| `short` | Smaller int (‚Č•16 bits) | `short int x;` or just `short x;` |
| `long` | Larger int (‚Č•32 bits) | `long x;` |
| `long long` | Even larger (‚Č•64 bits) | `long long x;` |
| `unsigned` | No negative values, double the positive range | `unsigned int x;` |
| `signed` | Explicit signed (default for int) | `signed char c;` |

K&R's key point: the exact sizes are machine-dependent. What's guaranteed is only
the *minimum* sizes and that `short ‚Č§ int ‚Č§ long ‚Č§ long long`.

Use `sizeof(type)` to check sizes on your machine:

```c
printf("char:   %zu bytes\n", sizeof(char));    // always 1
printf("int:    %zu bytes\n", sizeof(int));     // typically 4
printf("long:   %zu bytes\n", sizeof(long));    // 4 or 8
printf("double: %zu bytes\n", sizeof(double));  // typically 8
```

## Variables

A variable is a named storage location. Declare before use. Always.

```c
int count;          // declaration
count = 10;         // assignment

int total = 0;      // declaration + initialization (preferred)
char letter = 'A';
double pi = 3.14159265;
```

### Naming rules

- Must start with a letter or underscore
- Can contain letters, digits, underscores
- Case-sensitive (`count` ‚Č† `Count`)
- Don't start with `_` in practice ‚ÄĒ reserved for the system
- K&R convention: lowercase, short, descriptive. `n`, `i`, `sum`, `max_val`

### Scope

Where you declare a variable determines where it's visible:

```c
int global = 100;   // visible everywhere (avoid when possible)

int main(void) {
    int local = 42;  // visible only inside main

    {
        int inner = 7;  // visible only inside this block
    }
    // inner is gone here
}
```

### Constants

Two ways:

```c
#define MAX_SIZE 100         // preprocessor constant ‚ÄĒ no type, no semicolon
const int max_size = 100;   // typed constant ‚ÄĒ preferred in modern C
```

K&R uses `#define` throughout. Modern practice prefers `const` for type safety.
Use `#define` for values that need to work in preprocessor contexts (like array sizes in C89).

## Operators

### Arithmetic

| Operator | Meaning | Example | Result |
|----------|---------|---------|--------|
| `+` | Addition | `5 + 3` | `8` |
| `-` | Subtraction | `5 - 3` | `2` |
| `*` | Multiplication | `5 * 3` | `15` |
| `/` | Division | `5 / 3` | `1` (integer division truncates!) |
| `%` | Modulus (remainder) | `5 % 3` | `2` |

**Integer division truncates toward zero.** This is the most common gotcha for beginners. `5 / 3` is `1`, not `1.666...`. If you want the decimal result, at least one operand must be a float:

```c
5 / 3       // 1     (int / int = int)
5.0 / 3     // 1.666 (double / int = double)
(double)5/3 // 1.666 (cast forces floating-point division)
```

### Relational and Logical

| Operator | Meaning | Note |
|----------|---------|------|
| `==` | Equal to | NOT `=` (that's assignment) |
| `!=` | Not equal to | |
| `<` `>` `<=` `>=` | Comparisons | Work on numbers and chars |
| `&&` | Logical AND | Short-circuits: if left is false, right is not evaluated |
| `\|\|` | Logical OR | Short-circuits: if left is true, right is not evaluated |
| `!` | Logical NOT | `!0` is `1`, `!anything_else` is `0` |

In C, **zero is false, everything else is true.** There is no boolean type in K&R C (C99 added `_Bool`).

### Assignment

| Operator | Equivalent | Meaning |
|----------|-----------|---------|
| `=` | ‚ÄĒ | Assign |
| `+=` | `x = x + n` | Add and assign |
| `-=` | `x = x - n` | Subtract and assign |
| `*=` | `x = x * n` | Multiply and assign |
| `/=` | `x = x / n` | Divide and assign |
| `%=` | `x = x % n` | Modulus and assign |

### Increment / Decrement

```c
int x = 5;
x++;    // x is now 6 (post-increment: returns old value, then increments)
++x;    // x is now 7 (pre-increment: increments, then returns new value)
```

The difference matters when used in expressions:

```c
int a = 5;
int b = a++;  // b = 5, a = 6  (a's old value assigned to b)
int c = ++a;  // c = 7, a = 7  (a incremented first, then assigned)
```

K&R advice: prefer `++i` over `i++` when either would work. Avoid using increment
operators inside complex expressions ‚ÄĒ it's clever but unreadable.

### Bitwise

| Operator | Meaning | Example |
|----------|---------|---------|
| `&` | AND | `0b1100 & 0b1010` ‚Üí `0b1000` |
| `\|` | OR | `0b1100 \| 0b1010` ‚Üí `0b1110` |
| `^` | XOR | `0b1100 ^ 0b1010` ‚Üí `0b0110` |
| `~` | NOT (complement) | `~0b1100` ‚Üí `0b0011` (inverts all bits) |
| `<<` | Left shift | `1 << 3` ‚Üí `8` |
| `>>` | Right shift | `8 >> 2` ‚Üí `2` |

These operate on the individual bits of integers. You'll use them for flags,
masks, and low-level hardware manipulation.

## Type Conversion

C converts types automatically when mixing them in expressions. The rule:
narrower types promote to wider types.

```
char ‚Üí int ‚Üí long ‚Üí float ‚Üí double
```

```c
int i = 5;
double d = 3.2;
double result = i + d;  // i promoted to double, result is 8.2
```

Explicit casts when you want to force a conversion:

```c
int x = 7, y = 2;
double ratio = (double)x / y;  // 3.5, not 3
```

**Warning:** converting a wider type to a narrower type can lose data silently:

```c
int big = 100000;
short small = big;  // may overflow ‚ÄĒ no warning by default
```

## Putting It Together

A small program that uses everything above ‚ÄĒ temperature conversion from K&R:

```c
#include <stdio.h>

int main(void) {
    double fahr, celsius;
    int lower = 0;
    int upper = 300;
    int step = 20;

    printf("  F       C\n");
    printf("-----  ------\n");

    fahr = lower;
    while (fahr <= upper) {
        celsius = (5.0 / 9.0) * (fahr - 32.0);
        printf("%5.0f  %6.1f\n", fahr, celsius);
        fahr += step;
    }

    return 0;
}
```

Note `5.0 / 9.0` instead of `5 / 9`. The latter is integer division and gives `0`.
This is exactly the bug K&R warns about in Chapter 1.

## Common Mistakes

- **`=` vs `==`** ‚ÄĒ `if (x = 5)` assigns 5 to x and is always true. You meant `if (x == 5)`.
- **Integer division** ‚ÄĒ `5 / 3` is `1`, not `1.67`. Use floats if you need decimals.
- **Uninitialized variables** ‚ÄĒ local variables in C have garbage values by default. Always initialize.
- **Overflow** ‚ÄĒ `int` has limits (~¬Ī2 billion). Going past them wraps around silently for unsigned, or is undefined for signed.
- **Char signedness** ‚ÄĒ `char` may be signed or unsigned depending on the platform. Don't assume.
